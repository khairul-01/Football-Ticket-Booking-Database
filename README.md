# Football Ticket Booking System Database

This is database design and SQL queries for Football ticket booking system.  
Here an ERD diagram has been designed to show cardinality relationship of three tables - Users, Bookings, Matches. Here bookings table to avoiding many to many relation between users and matches. Thus we can understand primary key, foreign key and referential integrity.

Here sql has been written to create database, those three tables and insert sample values.

And around seven Business Logic has been done using JOIN variants, subqueries, aggregations, pattern matching, null handling, and pagination.
Complex sql queries like inner joni, left join/ full join, subqueries, aggregations like avg(), some function like round, aliasing, patter matching using like or ilike, null handling, and pagination with limit and offset.
